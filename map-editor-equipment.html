🎯 概要

設備カテゴリでは、信号扱いに必要な装置を配置します。 配線編集が完了し、軌道回路・駅・ホームが揃ったあとに進めるカテゴリです。

左サイドバーの設備カテゴリにある子モードは次のとおりです。

  • てこ — 信号扱い係が操作するスイッチ（3 種類）
  • 進路 — 発点てこから着点てこまでの 1 本の道筋
  • 信号機 — 場内・出発・閉塞・入換の 4 種類
  • 出発ボタン — ホーム端に置く発車合図ボタン

🎚 てこ

てこは信号扱い係（プレイヤー）が操作するスイッチで、発点てこと着点てこの組合せで進路を開通させます。 遊び方の詳細は 遊び方ガイドのてこの章 を参照してください。

てこの 3 種類

てこモードは 3 種類のサブモードを切り替えながら配置します。

サブモード役割選択時の色
本線用発点本線の進路を開通させる発点てこ黄色
入換用発点構内入換進路の発点てこ緑色
着点進路の終点（本線・入換ともに共通）—
用語の対応：本線用発点は「本線進路の発点てこ」、入換用発点は「入換進路の発点てこ」を指します。 着点てこは本線用と入換用で共通で、進路登録時にどちらの発点てこと組み合わさるかで意味が決まります。

配置方法

  1. てこモード（本線用発点・入換用発点・着点 のいずれかのサブモード）を選ぶ
  2. 軌道回路上のてこを置きたい位置をクリック
  3. 右側サイドバーでてこの名称（例: 1L, A）を確認・微調整 → 登録
  4. 進路を組むときに「この発点てこから、この着点てこへ」というペアで参照される

🛤 進路

進路は「ある発点てこから、ある着点てこまで」の 1 本の道筋を、通過する軌道回路の連なりとして登録したものです。 プレイヤーがてこを 2 つ操作したとき、この進路定義に合致するものがあれば開通します。

進路の登録手順

  1. 進路モードに入る
  2. 始点となる発点てこを選択（本線用・入換用どちらでも可）
  3. 終点となる着点てこを選択
  4. 進路が通過する軌道回路を順にクリックして連ねる
  5. サイドバーの登録ボタンで進路を確定 → Ctrl+S で保存
進路名は自動生成：進路の名前（ID）は、発点てこ名と着点てこ名を連結して自動的に決まります。 たとえば発点てこ 1L と着点てこ A から登録した進路は 1LA という ID になります。 ユーザーが進路名を手入力する必要はありません。

進路に含める軌道回路の選び方（重要）

進路として連ねた軌道回路は、その進路を守る信号機（場内・出発・入換）が「ここから先は自分の管理範囲」と宣言する区間になります。 信号機は管理範囲内に列車が在線している間は赤を示し、列車を進入させません。

このため、発点てこ・信号機が置かれている軌道回路は、進路に含めません。含めてしまうと、列車がてこ・信号機の位置に達した瞬間に信号機が自分自身を守って赤になり、列車がその場で立ち往生します。

原則：進路の最初の軌道回路は、発点てこ・信号機より進行方向の先にある軌道回路です。
発点てこと信号機をどこに置くかで、進路に含めるべき軌道回路の範囲が変わります。配線時に「ここの軌道回路で信号を扱う」と決め、てこと信号機をその手前側、進路の連ねをその先から開始するのが基本形になります。

実例: 軌道回路 TC_A → TC_B → TC_C と並んでおり、TC_A に発点てこを置き、TC_A と TC_B の境界に場内信号機を置き、TC_C に着点てこを置く場合、進路には TC_B, TC_C を連ねます。TC_A は進路に含めません。着点てこのある TC_C は進路に含めます（着点てこは進路の終端を示す目印で、その軌道回路は進路範囲の内側です）。

競合進路

交差する軌道回路を共有する 2 つの進路は同時に開通できません。エディタが自動的に競合関係を計算し、ゲーム実行時に「先に開通している進路がある」とログ表示します。 ユーザーが競合一覧を手書きで定義する必要はありません。

🚥 信号機

信号機は進路の状態に応じて自動で現示を変える表示装置です。 プレイヤーが信号機を直接操作することはなく、進路を開通させると進行現示に切り替わります。

信号機の種類

種類役割
場内信号機駅構内への進入を制御
出発信号機駅からの出発を制御
閉塞信号機駅間の閉塞区間（駅と駅の間）への進入を制御
入換信号機構内入換用

設置方法（場内・出発・入換）

場内・出発・入換信号機は、その信号機が「どの進路の現示を制御するか」を 担当進路として指定する必要があります。 担当進路は、進路登録時と同じく発点てこ → 着点てこのペアで自動検索されます。

  1. 信号機モードに入り、種類（場内 / 出発 / 入換）を選ぶ
  2. マップ上をクリックして信号機の位置を決める
  3. サイドバーで発点てこと着点てこを 1 つずつ選ぶ
  4. エディタが該当する進路を自動検索して「担当進路」として紐づける
  5. 登録ボタンで反映 → Ctrl+S で保存

設置方法（閉塞信号機）

閉塞信号機は担当進路を持たず、代わりに防護する軌道回路（複数可）を直接指定します。

Shift を押しながらクリック：閉塞信号機の防護軌道回路は Shift を押しながらクリックで選択します（複数選択も可）。 通常クリックでは選択モードが切り替わらず、初見で気づきにくいのでエディタ側でも黄色いヒントが出ます。

現示の制限（UGC マップ）

UGC マップでは信号現示は青（進行）・黄（注意）・赤（停止）の自動切替が基本です。 警戒・減速・抑速といった現示の細かな使い分けはエディタ側では設定できません。

🛑 出発ボタン

出発ボタンはホーム端に置く、プレイヤーが押すボタンです。押すと該当列車が発車合図を受けて動き出します。

番線ベース動作（UGC マップ）

UGC マップでは出発ボタンは番線を基準に動作します。 ボタンに「どの駅のどの番線を担当するか」を登録しておくと、その番線に在線中の列車に発車合図を出します。

配置と登録

  1. 出発ボタンモードに入る
  2. ホーム端の位置をクリックして配置
  3. 右側サイドバーで担当する駅を選び、続いてその駅の番線を選ぶ（番線一覧は番線マスタから自動生成）
  4. 登録ボタンで反映、Ctrl+S で CSV へ保存
公式マップとの違い：公式マップでは出発ボタンに進路名や選別装置 ID を直接結びつける方式を採用していますが、UGC では駅×番線の指定だけで動く番線ベースに簡略化されています。
